# 组会说明稿（当前纯步长版本）

## 1. 这个项目要解决什么问题

目标是追踪矩阵 `A` 的 `epsilon`-伪谱等高线：

\[
\sigma_{\min}(zI-A)=\epsilon
\]

也就是在复平面里沿着这条曲线走一圈。

如果完全用传统数值方法，常见做法是：

- 用局部几何给出切向预测
- 然后频繁做精确 SVD 或 Newton 校正

瓶颈通常不在“切向方向怎么写”，而在：

- 你为了保证不跑偏，需要反复做昂贵校正
- 如果步长控制不好，总步数也会很多

所以当前项目的核心思想不是让神经网络直接生成整条轮廓，而是：

- **白盒几何负责方向**
- **神经网络负责步长控制**
- **必要时才做精确刷新 / projection**

这是一个典型的“数值算法 + 学习控制器”的混合框架。

---

## 2. 当前主线流程

主线只有四步：

1. 随机生成矩阵与起点，构造离线任务
2. 用高精度专家给每个状态打 `ds_expert`
3. 训练一个轻量级 MLP，学习从局部状态到步长
4. 推理时让网络输出步长，白盒追踪器负责执行

当前实际使用的核心脚本是：

- `scripts/generate_large_dataset.py`
- `scripts/train_from_dataset.py`
- `scripts/run_tracking.py`
- `scripts/benchmark_nn_vs_newton.py`

---

## 3. 数据集怎么生成

### 3.1 一个训练任务怎么定义

每个任务不是手工指定固定 `epsilon`，而是这样生成：

1. 随机抽一个矩阵类型
2. 按尺寸生成矩阵 `A`
3. 在谱中心附近随机采一个复平面点 `z_random`
4. 计算
   \[
   \epsilon = \sigma_{\min}(z_{random}I-A)
   \]
5. 把“经过这个随机点的等高线”当成当前追踪任务

所以训练数据、推理任务、benchmark 任务三者是一致的。

### 3.2 专家标签怎么来

当前标签由 `src/train/expert_solver.py` 生成。

专家做的事情是：

- 用 full triplet RK4 推进 `(z,u,v)`
- 对候选点做局部 projection 或回退投影
- 通过 backtracking 调整 teacher step
- 最终输出一个高质量的 `ds_expert`

也就是说，专家标签不是分类标签，而是**纯回归步长标签**。

### 3.3 为什么还要 DAgger

如果只在“理想专家轨迹”上训练，网络见到的状态太干净。

一旦推理时状态有扰动，网络可能不会恢复。

所以数据生成还会做 DAgger 风格增强：

- 对专家轨迹上的 `(z,u,v)` 加小扰动
- 把扰动状态重新投影回 contour 附近
- 再查询专家给出恢复步长

这样训练集里就同时包含：

- 标准专家轨迹样本
- 偏离轨道后的恢复样本

## 4. 神经网络学什么

### 4.1 输入：8 维局部状态特征

当前输入不是矩阵原始条目，也不是整段轨迹，而是 8 维局部状态：

1. `|sigma_approx - epsilon|`
2. `||Mv - epsilon u||`
3. `|u^*v|`
4. `|epsilon|`
5. `matrix_scale`
6. `|z| / matrix_scale`
7. `prev_ds`
8. `prev_applied_projection`

可以理解成三类信息：

- 当前 triplet 漂不漂
- 当前任务尺度和位置
- 刚才一步走得激不激进

### 4.2 输出：只有步长 `ds`

当前 `NNController` 只输出一个标量：

- `ds`

没有 restart head，也没有分类分支。

### 4.3 损失函数

当前训练目标非常简单：

- 预测 `ds_expert`
- 用 log-space MSE 做监督

这样做的原因是步长本身跨数量级变化较大，直接在线性尺度上做 MSE 不够稳。

## 5. 当前推理算法到底怎么跑

这是最关键的部分。

当前主线推理并不是“每一步都重新精确求完整三元组”。

benchmark 和 `run_tracking` 默认走的是：

- `FAST_TANGENT_TRACKER_KWARGS`
- `integration_method='tangent'`

也就是一条 **fast tangent tracker** 路线。

### 5.1 初始化

推理开始时先做一次精确初始化：

- 给定 `z0`
- 求出对应的最小奇异值三元组 `(u,v)`
- 把当前状态设为 `(z,u,v)`

### 5.2 每一步主循环

每一步可以概括成：

1. 从当前 `(z,u,v)` 提取 8 维特征
2. `NNController` 预测原始步长 `ds`
3. `AdaptiveInferenceController` 对 `ds` 做稳定化包装
4. 用白盒切向方向推进 `z`
5. 用当前 triplet 在新点上估计误差
6. 根据误差决定：
   - 直接接受
   - 延迟 projection
   - 局部 projection
   - 必要时做 exact triplet refresh
   - 或者 backtracking 缩步

### 5.3 ODE 在这里提供什么

ODE 负责给出沿等高线的**方向**，核心是切向：

\[
rac{dz}{ds} = irac{v^*u}{|v^*u|}
\]

所以：

- 方向是白盒几何给的
- 神经网络不负责决定朝哪走
- 神经网络只负责“走多远”

### 5.4 `u,v` 如何维护

当前快路径里真正昂贵的东西是精确刷新，而不是切向本身。

所以主线策略是：

- 很多步先只推进 `z`
- 尽量复用当前 `(u,v)` 做近似判定
- 只有近似误差变大时才做局部投影或精确刷新

因此真正快的原因不是“网络前向很快”，而是：

- 少做 SVD
- 少做完整精确校正
- 用更好的步长控制减少总步数

## 6. 为什么“只学步长”仍然有效

这个问题很关键。

一个常见质疑是：

> 传统算法的 bottleneck 是 SVD 很慢，但算梯度 / 切向本身接近 O(1)。
> 你的算法也没有把 SVD 完全拿掉，只是多了一个步长预测，为啥还能有效？

答案是：**因为它优化的是“什么时候需要昂贵操作、总共需要走多少步”，而不是优化单次切向公式。**

更具体地说，它的收益来自三点：

### 6.1 更少的总步数

如果步长策略足够好：

- 平滑区能大胆走大步
- 危险区能及时缩步

那么同一条 contour 可以用更少的 accepted steps 跑完。

### 6.2 更少的不必要精确校正

传统保守算法常常为了安全而高频校正。

而当前 fast tracker 会先做近似判定：

- 如果当前 triplet 还可信，就继续走
- 只有真的快不可信了才触发 correction

所以精确 SVD / projection 的调用频率会下降。

### 6.3 更少的失败回退

如果步长选得不好，会出现：

- backtracking 次数增加
- projection 次数增加
- contour 上抖动更明显
- 闭合前走很多冤枉路

网络学到的是一种经验性控制规律：

- 什么局部状态下应该保守
- 什么局部状态下可以激进

这个规律虽然不改变单次 SVD 的复杂度，但能显著改变**整个 rollout 的计算结构**。

所以核心收益不是：

- “SVD 从慢变快”

而是：

- “昂贵步骤发生得更少”
- “总 rollout 更短”
- “整体 wall-clock 更优”

## 7. 与传统 Newton predictor-corrector 的关系

当前 baseline 是 `src/baselines/newton_predictor_corrector.py`。

它的特点是：

- 用切向做 predictor
- 用 Newton normal corrector 回到 contour
- 通过 halving / line search 保证收敛

这是一个很合理的经典基线。

而当前 NN+ODE 方法的区别是：

- Newton baseline 主要靠数值校正策略保证稳定
- NN+ODE 方法把“步长控制”单独学出来
- 快路径允许更多近似复用和延迟纠偏

所以它们比较的重点不是数学正确性，而是：

- 相同任务下谁更快
- 谁用更少步完成
- 谁更少依赖高频昂贵校正

---

## 8. 当前版本最重要的结论

一句话概括：

> 当前仓库已经从“步长 + restart 双头控制器”收缩为“纯步长控制器 + fast tangent tracker”。

因此当前系统的真实结构是：

- **方向：白盒 ODE / 几何给出**
- **步长：神经网络预测**
- **稳定性：projection / backtracking / 低频精确刷新保证**

这是现在最干净、也最符合代码实际行为的表述。
